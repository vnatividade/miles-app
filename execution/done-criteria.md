# Done Criteria — miles-app

## Purpose

Define when a task can be considered truly complete.

---

# Minimum Done Criteria

A ticket is only DONE when:

- implementation is completed
- code compiles or runs correctly
- acceptance criteria are satisfied
- ticket status is updated
- PR is opened or updated
- relevant documentation is updated
- handoff notes exist

---

# Documentation Rule

If implementation changes architecture, workflows, or operational behavior:

Repository documentation must be updated.

---

# Handoff Rule

Every completed ticket must leave enough context for another agent to continue safely.

---

# Anti-Fake-Done Rule

A task is NOT done if:

- code exists but was not documented
- implementation changed scope without ticket updates
- next steps are unclear
- blockers were not registered
- PR lacks context
- architectural impact was ignored

---

# MVP Discipline

Done does not mean perfect.

Done means:
- aligned with MVP scope
- operational
- understandable
- evolvable
