# Skill — Prepare PR Handoff

## Purpose

Prepare operationally useful pull requests and handoff notes.

---

# Required Inputs

- Linear ticket
- files changed
- implementation summary
- pending items
- risks

---

# PR Requirements

The PR must contain:

- ticket reference
- context
- included scope
- excluded scope
- files changed
- risks
- operational notes
- handoff

---

# Handoff Requirements

Every handoff should explain:

## Completed

What was implemented.

## Pending

What still needs work.

## Risks

Technical or operational concerns.

## Suggested Next Ticket

Logical next step.

---

# Important Rule

Future agents should not need to reverse engineer implementation decisions.

---

# Expected Outputs

- completed PR template
- operational handoff
- next execution recommendation
