# Skill — Update Project Docs

## Purpose

Keep repository documentation aligned with implementation and operational changes.

---

# When To Use

Use this skill when:

- architecture changes
- workflows change
- operational behavior changes
- contracts change
- important implementation decisions are made

---

# Required Context

Read:

```txt
product/
architecture/
execution/
agents/
relevant Linear ticket
```

---

# Documentation Principles

## 1. Preserve Clarity

Documentation should remain understandable for future agents.

## 2. Preserve Traceability

Changes should explain:
- why something changed
- what changed
- operational implications

## 3. Avoid Stale Docs

Repository documentation must reflect actual implementation.

---

# Important Rule

Documentation is part of the implementation, not a separate optional task.

---

# Expected Outputs

- updated markdown docs
- architecture adjustments
- workflow updates
- operational notes
