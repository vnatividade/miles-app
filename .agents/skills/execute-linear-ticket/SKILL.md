# Skill — Execute Linear Ticket

## Purpose

Execute a Linear ticket safely and consistently.

---

# Required Context

Before execution:

1. Read `AGENTS.md`
2. Read relevant `product/` documents
3. Read relevant `architecture/` documents
4. Read `execution/` rules
5. Read assigned Linear ticket

---

# Execution Flow

```txt
Read Ticket
→ Understand Scope
→ Identify Expected Files
→ Create/Use Branch
→ Implement
→ Validate Scope
→ Update Docs
→ Prepare PR
→ Prepare Handoff
```

---

# Mandatory Checks

Before implementation confirm:

- ticket scope is clear
- acceptance criteria exist
- no unrelated work is included
- architecture constraints are understood

---

# Important Rules

- execute one ticket at a time
- avoid hidden scope expansion
- preserve MVP discipline
- document important decisions
- preserve operational traceability

---

# Expected Outputs

- implementation
- updated documentation if needed
- PR draft or PR notes
- handoff notes
- suggested next ticket
