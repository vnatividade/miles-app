# Agentic Maturity Model — miles-app

## Purpose

Define the evolution stages of the miles-app agentic operating system.

---

# Level 0 — Prompt Assisted

Characteristics:
- isolated prompts
- no persistent context
- no operational continuity

---

# Level 1 — Contextual Repository

Characteristics:
- AGENTS.md
- architecture docs
- product docs
- workflow docs

Current status: completed.

---

# Level 2 — Operational Execution

Characteristics:
- Linear integration
- tickets
- PR templates
- handoff rules
- execution workflows

Current status: completed.

---

# Level 3 — Reusable Skills

Characteristics:
- reusable implementation workflows
- technical execution patterns
- modular skills
- operational hooks

Current status: in progress.

---

# Level 4 — Coordinated Multiagent System

Characteristics:
- specialized subagents
- orchestrated execution
- review agents
- product guardians
- coordinated PR flow

Current status: in progress.

---

# Level 5 — Autonomous Operational Platform

Characteristics:
- hooks automation
- dynamic task decomposition
- self-improving workflows
- automatic quality gates
- continuous operational learning

Current status: future.

---

# Important Rule

Autonomy should only increase after:
- operational clarity
- traceability
- safe workflows
- understandable handoffs

Never optimize for autonomy before governance.
